# VivaTrace

VivaTrace — локальный AI-first контур адаптивного обучения для неоднородной университетской группы.

Студент выполняет задание по теме пары. В заданиях с однозначным ключом правильность каждого пункта считает детерминированный предметный grader; в открытых письмах и эссе содержание оценивает локальная LLM. Затем локальная LLM формирует вопросы и выбирается одна из двух веток:

```text
Задание выполнено правильно
    → viva подтверждает самостоятельное понимание
    → задание на перенос навыка в новый контекст

Задание выполнено неправильно
    → диагностические вопросы локализуют пробел
    → объяснение + разобранный пример + повторная практика
```

После завершения попытки та же локальная LLM анализирует последние результаты группы и предлагает преподавателю фокус и план следующего занятия.

## Главное об AI-контуре

- в быстром режиме используется скачанная `Qwen2.5-3B-Instruct Q4_K_M`, а 7B-модель доступна как углублённый режим;
- inference выполняет локальный `llama.cpp` на `127.0.0.1`;
- API-ключи, OpenAI и интернет при работе приложения не нужны;
- продукт не имеет облачного fallback: если модель отсутствует, смысловая проверка честно блокируется;
- 23 структурированных задания проверяются скриптом по нормализованному ключу с допустимыми эквивалентами; LLM не может переопределить правильный `can't`, `must have`, сокращённую форму или другой точный ответ;
- каждый этап сохраняет модель, SHA-256 весов, trace ID, длительность и расход токенов;
- оценка текущего ответа показывается отдельно от накопленного BKT-освоения.

В техническом журнале видно разделение ответственности: предметный ключ отвечает за воспроизводимый балл закрытого задания, а локальная LLM — за семантическую оценку открытого ответа, вопросы Viva, анализ объяснений, персональный маршрут и план занятия.

## Оценка качества grader

Adversarial benchmark содержит правильные, частичные, пустые и специально перепутанные ответы по временам, модальным глаголам, артиклям, conditionals, passive voice и gerund/infinitive.

```powershell
.venv\Scripts\python.exe scripts\evaluate_grader.py
```

Текущий результат на 24 сценариях: `exact score accuracy = 100%`, `false accepts = 0`, `false rejects = 0`. Benchmark выполняется и внутри pytest.

## Предметы и задания

При первом запуске база получает:

- исходное задание по валидации ML-модели;
- 30 заданий английского языка уровня B2: по три неповторяющихся варианта базового, среднего и продвинутого уровня для каждой темы:
  - Present Perfect и Past Simple;
  - First и Second Conditional;
  - Passive Voice;
  - Reported Speech;
  - modal verbs of deduction;
  - relative clauses;
  - gerund и infinitive;
  - articles;
  - linking words;
  - formal email.

У каждого задания есть предметная рубрика, эталон, критерии и типичные ошибки. Локальная база правил связывает результат с конкретным объяснением и открытым учебным источником. Преподаватель может создавать новые задания и редактировать рубрику.

В кабинете преподавателя кнопка «Добавить» открывает простой конструктор без JSON: можно добавить вариант к существующей теме или создать новую тему вместе с первым заданием. Для существующей темы предмет, навыки и номер варианта подставляются автоматически; преподаватель вводит условие, эталон и критерии обычным текстом.

Полный результат Viva сохраняется в SQLite и восстанавливается после навигации или перезапуска приложения. В личном кабинете студент видит выполненные задания, число завершённых проверок, средний балл и навыки, которые стоит повторить. Кабинет преподавателя отделяет проверяемые факты — несовпадения с рубрикой и баллы Viva — от свободного текста модели; повреждённая рекомендация не попадает в интерфейс.

## Установка на Windows

```powershell
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
powershell -ExecutionPolicy Bypass -File scripts\setup_local_llm.ps1
.venv\Scripts\streamlit.exe run app.py
```

Скрипт установки:

1. загружает закреплённую Windows-сборку `llama.cpp`;
2. загружает Qwen2.5-3B для быстрого интерактивного контура и два сегмента Qwen2.5-7B для углублённого режима;
3. проверяет SHA-256 runtime и обеих моделей;
4. сохраняет манифест версии;
5. не требует повторной загрузки при следующем запуске.

При первой проверке ответа приложение само запускает `llama-server.exe` в скрытом режиме. После этого интернет можно отключить.

## Docker

После загрузки модели той же командой:

```bash
docker compose up --build
```

Compose запускает официальный локальный `llama.cpp:server`, монтирует GGUF только для чтения и подключает к нему приложение во внутренней Docker-сети.

## Что сохраняется

SQLite хранит:

- три демонстрационных аккаунта;
- задания и рубрики;
- исходное решение студента;
- оценку задания и выбранную ветку;
- вопросы и оценки viva;
- BKT-состояние навыков;
- персональное следующее задание;
- LLM-рекомендацию преподавателю;
- полный технический аудит локальных LLM-вызовов.

В пульсе группы учитывается только последняя попытка каждого студента по выбранной теме — даже если студенты получили разные варианты. Предзаполненных результатов нет.

## Проверки

```bash
pytest -q
ruff check .
python scripts/run_experiment.py
python scripts/run_local_llm_eval.py
```

`run_local_llm_eval.py` выполняет живой тест весов: правильный и бессмысленный ответы, маршрутизацию viva/diagnostic, три негативных viva-кейса и три регрессии по артиклям из пользовательского тестирования. Он сохраняет метрики и полные локальные трассы в `artifacts/local_llm_eval.json`.

## Архитектура

```text
Ответ студента
    ↓
Local LLM: оценка задания + выбор viva/diagnostic
    ↓
Local LLM: 2 персональных viva/diagnostic вопроса
    ↓
Local LLM: независимая оценка каждого ответа
    ↓
BKT: накопленная модель освоения (не путать с текущей оценкой)
    ↓
Local LLM: transfer/remediation + план следующей пары по группе
    ↓
SQLite: результаты + LLM trace + SHA-256 модели
    ↓
Кабинет студента / пульс преподавателя
```

## Ограничения MVP

- локальная модель всё ещё может ошибаться, поэтому её качество нужно измерять на размеченном наборе ответов;
- первая загрузка обоих наборов весов требует около 6,8 ГБ и дополнительного места для сегментов 7B-модели;
- первый запуск модели занимает время, последующие вызовы используют уже загруженный процесс;
- на CPU быстрые вызовы обычно занимают десятки секунд; `LOCAL_LLM_QUALITY_MODE=1` повышает качество ценой заметно большей задержки;
- голосовой интерфейс, авторизация и LMS-интеграция пока не реализованы;
- образовательный эффект требует пилота с реальными студентами.
