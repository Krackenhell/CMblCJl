# VivaTrace

VivaTrace — локальный AI-first контур адаптивного обучения для неоднородной университетской группы.

Студент выполняет задание по теме пары. В заданиях с однозначным ключом правильность каждого пункта считает детерминированный предметный grader; в открытых письмах и эссе содержание оценивает локальная LLM. Затем локальная LLM формирует вопросы и выбирается одна из двух веток:

```text
Задание выполнено правильно
    → viva подтверждает самостоятельное понимание
    → задание на перенос навыка в новый контекст

Задание выполнено неправильно
    → диагностические вопросы локализуют пробел
    → объяснение + разобранный пример + повторная практика
```

После завершения попытки та же локальная LLM анализирует последние результаты группы и предлагает преподавателю фокус и план следующего занятия.

В отдельном режиме «Практическая миссия» студент применяет правило в коротком
ролевом диалоге: например, разыскивает багаж, объясняет офисное происшествие или
помогает новому студенту. Код проверяет наличие обязательных языковых конструкций,
а локальная LLM играет персонажа и оценивает коммуникативный результат. В MVP есть
10 миссий — по одной на каждую тему B2.

В отдельном режиме «Голосовая Viva» студент разговаривает с ботом по теме урока.
Постоянный WebSocket передаёт PCM-аудио в обе стороны, микрофон остаётся активным
во время ответа и позволяет перебить бота. Whisper.cpp и Silero VAD локально получают
транскрипт, LanguageTool и предметные правила независимо проверяют структуру, Qwen
ведёт диалог и оценивает смысл, а Windows SAPI озвучивает ответ. В кабинет
преподавателя поступают транскрипты, темп, паузы, fillers и языковые оценки.

## Главное об AI-контуре

- в быстром режиме используется скачанная `Qwen2.5-3B-Instruct Q4_K_M`, а 7B-модель доступна как углублённый режим;
- inference выполняет локальный `llama.cpp` на `127.0.0.1`;
- API-ключи, OpenAI и интернет при работе приложения не нужны;
- продукт не имеет облачного fallback: если модель отсутствует, смысловая проверка честно блокируется;
- 71 структурированное задание проверяется предметным grader: точные формы — нормализованным ключом, а сложные преобразования — переиспользуемыми компонентами правила с частичным баллом; LLM не может обнулить почти правильное решение или переопределить правильный `can't`, `must have` и сокращённую форму;
- каждый этап сохраняет модель, SHA-256 весов, trace ID, длительность и расход токенов;
- голосовой режим также не использует облако: Whisper, Silero VAD, LanguageTool,
  Qwen и синтез речи работают на устройстве;
- замечание миссии показывается только с буквальной цитатой ответа; переписывание
  ответа и галлюцинации об артиклях отбрасываются до выставления результата;
- низкая уверенность смысловой Viva не маскируется: ответ помечается для просмотра
  преподавателем;
- оценка текущего ответа показывается отдельно от освоения, которое объединяет историю, исходное задание и оба ответа Viva.

В техническом журнале видно разделение ответственности: предметный ключ отвечает за воспроизводимый балл закрытого задания, а локальная LLM — за семантическую оценку открытого ответа, вопросы Viva, анализ объяснений, персональный маршрут и план занятия.

## Оценка качества grader

Adversarial benchmark содержит правильные, частичные, пустые и специально перепутанные ответы по временам, модальным глаголам, артиклям, conditionals, passive voice и gerund/infinitive.

```powershell
.venv\Scripts\python.exe scripts\evaluate_grader.py
```

Текущий результат на 26 сценариях: `exact score accuracy = 100%`, `false accepts = 0`, `false rejects = 0`. В набор входят допустимые перестройки Relative Clauses и частичная компонентная оценка; benchmark выполняется и внутри pytest.

Отдельный benchmark семантической Viva хранит экспертные интервалы для правильных,
частичных и adversarial-ответов. Knowledge-tracing эксперимент сравнивает BKT с тремя baseline
и считает Brier, log loss, ECE и accuracy. Команды, протокол и честные результаты описаны в
[ML_EXPERIMENTS.md](docs/ML_EXPERIMENTS.md).

## Предметы и задания

При первом запуске база получает:

- исходное задание по валидации ML-модели;
- 90 заданий английского языка уровня B2: по девять неповторяющихся вариантов для каждой из десяти тем, с базовым, средним и продвинутым уровнями:
  - Present Perfect и Past Simple;
  - First и Second Conditional;
  - Passive Voice;
  - Reported Speech;
  - modal verbs of deduction;
  - relative clauses;
  - gerund и infinitive;
  - articles;
  - linking words;
  - formal email.

У каждого задания есть предметная рубрика, эталон, критерии и типичные ошибки. Локальная база правил связывает результат с конкретным объяснением и открытым учебным источником. Преподаватель может создавать новые задания и редактировать рубрику.

В кабинете преподавателя кнопка «Добавить» открывает простой конструктор без JSON: можно добавить вариант к существующей теме или создать новую тему вместе с первым заданием. Для существующей темы предмет, навыки и номер варианта подставляются автоматически; преподаватель вводит условие, эталон и критерии обычным текстом.

Полный результат Viva и практических миссий сохраняется в SQLite и восстанавливается после навигации или перезапуска приложения. В личном кабинете студент видит выполненные задания, число завершённых проверок, миссии, навыки для повторения и прозрачный план интервального повторения. Кабинет преподавателя отделяет проверяемые факты — несовпадения с рубрикой, баллы Viva и подтверждённые цитаты миссий — от свободного текста модели; повреждённая рекомендация не попадает в интерфейс.

## Установка на Windows

```powershell
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
powershell -ExecutionPolicy Bypass -File scripts\setup_local_llm.ps1
powershell -ExecutionPolicy Bypass -File scripts\setup_local_voice.ps1
.venv\Scripts\streamlit.exe run app.py
```

Скрипт установки:

1. загружает закреплённую Windows-сборку `llama.cpp`;
2. загружает Qwen2.5-3B для быстрого интерактивного контура и два сегмента Qwen2.5-7B для углублённого режима;
3. проверяет SHA-256 runtime и обеих моделей;
4. сохраняет манифест версии;
5. не требует повторной загрузки при следующем запуске.

При первой проверке ответа приложение само запускает `llama-server.exe` в скрытом режиме. После этого интернет можно отключить.

`setup_local_voice.ps1` один раз устанавливает закреплённые whisper.cpp v1.9.1,
Whisper base.en, Silero VAD 6.2, LanguageTool 6.6 и переносимый Temurin JRE 17,
проверяет SHA-256 каждого архива и модели. При открытии режима приложение само
поднимает локальные WebSocket- и grammar-серверы.

## Docker

После загрузки модели той же командой:

```bash
docker compose up --build
```

Compose запускает официальный локальный `llama.cpp:server`, монтирует GGUF только для чтения и подключает к нему приложение во внутренней Docker-сети.

## Что сохраняется

SQLite хранит:

- три демонстрационных аккаунта;
- задания и рубрики;
- исходное решение студента;
- оценку задания и выбранную ветку;
- вопросы и оценки viva;
- объединённое состояние навыков по истории, заданию и Viva;
- персональное следующее задание;
- LLM-рекомендацию преподавателю;
- полный технический аудит локальных LLM-вызовов.
- диалоги и состояние практических миссий в отдельной таблице, не затрагивающей
  основной контур попыток.
- голосовые сессии и реплики: транскрипт, ответ бота, акустические метрики,
  предметные находки, оценки и LLM trace.

В пульсе группы учитывается только последняя попытка каждого студента по выбранной теме — даже если студенты получили разные варианты. Предзаполненных результатов нет.

## Проверки

```bash
pytest -q
ruff check .
python scripts/run_experiment.py
python scripts/run_local_llm_eval.py
python scripts/evaluate_viva_semantics.py
python scripts/evaluate_knowledge_tracing.py
python scripts/evaluate_missions.py --live-all
python scripts/smoke_local_voice.py "I studied in Kyoto, which attracts many visitors."
```

`run_local_llm_eval.py` выполняет живой тест весов: правильный и бессмысленный ответы, маршрутизацию viva/diagnostic, три негативных viva-кейса и три регрессии по артиклям из пользовательского тестирования. Он сохраняет метрики и полные локальные трассы в `artifacts/local_llm_eval.json`.

Текущая проверенная сборка: `84` unit/integration теста; предметный benchmark
`26/26`; semantic Viva `14/14` в экспертных диапазонах, без ложных принятий и
отклонений; mission-grounding `20/20` детерминированных и `11/11` живых сценариев.
Все цифры относятся к локальному eval-набору,
а не к доказанному образовательному эффекту.

## Архитектура

```text
Ответ студента
    ↓
Предметный grader: атомарные компоненты + воспроизводимый частичный балл
    ↓
Local LLM: 2 персональных viva/diagnostic вопроса одним batch-вызовом
    ↓
Local LLM + concept calibration: семантическая оценка каждого ответа
    ↓
Evidence fusion: история + исходное задание + оба ответа Viva
    ↓
Local LLM: transfer/remediation + план следующей пары по группе
    ↓
SQLite: результаты + LLM trace + SHA-256 модели
    ↓
Кабинет студента / пульс преподавателя

Практическая миссия (изолированный контур)
    → regex/contract signals + 1 локальный LLM-вызов на ход
    → фильтр дословных цитат и локальных исправлений
    → отдельная mission_attempts + обновление mastery только при успехе

Голосовая Viva (изолированный контур)
    → streaming PCM + client energy gate + barge-in по WebSocket
    → Whisper.cpp + Silero VAD → транскрипт
    → LanguageTool + предметные structural rules → независимые grammar facts
    → Qwen 3B → смысл, диалог и feedback; 7B только для неуверенного случая
    → Windows SAPI → голосовой ответ
    → voice_sessions / voice_turns → speaking-пульс преподавателя
```

Подробнее: [архитектура](docs/ARCHITECTURE.md), [сценарий защиты](docs/PITCH.md),
[исследование и roadmap](docs/RESEARCH_AND_ROADMAP.md),
[атрибуция](docs/REUSE_AND_ATTRIBUTION.md).

## Ограничения MVP

- локальная модель всё ещё может ошибаться, поэтому её качество нужно измерять на размеченном наборе ответов;
- первая загрузка обоих наборов весов требует около 6,8 ГБ и дополнительного места для сегментов 7B-модели;
- первый запуск модели занимает время, последующие вызовы используют уже загруженный процесс;
- на CPU быстрые вызовы обычно занимают десятки секунд; `LOCAL_LLM_QUALITY_MODE=1` повышает качество ценой заметно большей задержки;
- транспорт голосового режима полнодуплексный и поддерживает barge-in, но Whisper
  пока обрабатывает законченную реплику, а не выдаёт частичные слова во время речи;
- произношение и акцент не входят в балл: для честной оценки нужен отдельный
  forced alignment по фонемам;
- авторизация и LMS-интеграция пока не реализованы;
- банк миссий пока шаблонный; автоматическая генерация новых сценариев требует
  отдельной предметной валидации;
- образовательный эффект требует пилота с реальными студентами.

## Лицензия

Код VivaTrace распространяется по MIT. Qwen, llama.cpp, whisper.cpp, Silero VAD и
