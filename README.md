# Meaning («Смысл»)

**Адаптивный учебный тренажёр, который проверяет не только ответ студента, но и понимание, а затем помогает преподавателю увидеть общие пробелы группы.**

Проект подготовлен для Junior ML Contest 2026 в категории «AI в образовании».

## Какую проблему решает проект

В одной университетской группе студенты начинают обучение с разной подготовкой. Единый темп приводит к двум проблемам:

- студент с пробелами перестаёт понимать новые темы;
- сильный студент получает слишком простые задания и теряет интерес.

Обычный тест показывает выбранный или введённый ответ, но не всегда показывает, понимает ли студент правило. Преподаватель обычно замечает общий пробел только после контрольной работы.

Meaning («Смысл») замыкает учебный цикл:

```text
Задание по теме занятия
        ↓
Точная предметная проверка ответа
        ↓
Короткая проверка понимания
        ↓
Персональное объяснение или более сложная практика
        ↓
Применение знания в новой ситуации
        ↓
Общие пробелы группы и план следующего занятия
```

## Почему это не обычный чат с языковой моделью

Система разделяет ответственность:

- **предметный модуль** точно проверяет задания с однозначным ответом;
- **локальная языковая модель** объясняет найденную ошибку, задаёт два вопроса на понимание и подбирает следующий шаг;
- **независимые правила** перепроверяют важные грамматические конструкции;
- **кабинет преподавателя** объединяет только подтверждённые ошибки реальных профилей студентов.

Языковая модель не может самостоятельно изменить балл закрытого задания. Это защищает тренажёр от уверенных, но неверных оценок.

## Что уже реализовано

- 90 заданий английского языка уровня B2: 10 тем по 9 персональных вариантов;
- отдельные поля ответа для каждого пункта задания;
- частичный балл и допустимые варианты правильных ответов;
- короткая проверка понимания после задания;
- персональный разбор ошибки и повторная практика;
- практические миссии с переносом знания в новую ситуацию;
- голосовой диалог по теме занятия;
- личный кабинет студента: выполненные задания, частые ошибки, уровень, опыт и учебная серия;
- кабинет преподавателя: пробелы группы и предложение темы следующего занятия;
- добавление новых тем и заданий без редактирования JSON;
- локальный запуск основной системы без платного API;
- 100 автоматических проверок кода.

## Подтверждённые результаты

| Проверка | Результат |
|---|---:|
| Автоматические тесты | 100 из 100 |
| Предметная проверка | 26 из 26 сценариев |
| Смысловая проверка открытых ответов | 14 из 14 ответов в экспертном диапазоне |
| Проверка практических миссий | 20 из 20 точных и 11 из 11 живых сценариев |

Эти результаты относятся к локальным контрольным наборам. Они подтверждают работоспособность программного контура, но не заменяют пилот с реальными студентами.

## Быстрый запуск на Windows с локальной моделью

Понадобятся Windows 10/11, Python 3.11 или новее и около 8 ГБ свободного места для основной и быстрой моделей.

```powershell
git clone https://github.com/Krackenhell/CMblCJl.git
cd CMblCJl

python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt

powershell -ExecutionPolicy Bypass -File scripts\setup_local_llm.ps1
.venv\Scripts\streamlit.exe run app.py
```

Скрипт `setup_local_llm.ps1` автоматически:

1. скачивает закреплённую сборку `llama.cpp`;
2. скачивает Qwen 2.5 3B для быстрых ответов и Qwen 2.5 7B для углублённой проверки;
3. проверяет контрольные суммы файлов;
4. сохраняет сведения о версиях;
5. не загружает модели повторно при следующем запуске.

При первой проверке ответа приложение само запускает локальный сервер модели в скрытом режиме. После загрузки файлов основной тренажёр работает без интернета.

## Как подключить свою локальную модель

Подходит любая instruct-модель в формате GGUF, которую может запустить `llama.cpp`. Наиболее проверенный вариант — Qwen 2.5 7B Instruct Q4_K_M.

1. Положите файл модели, например, в `models/my-model.gguf`.
2. Убедитесь, что `llama-server.exe` находится в `tools/llama/`.
3. Перед запуском задайте путь и название модели:

```powershell
$env:LOCAL_LLM_MODEL_PATH="models/my-model.gguf"
$env:LOCAL_LLM_FAST_MODEL_PATH="models/my-model.gguf"
$env:LOCAL_LLM_MODEL="my-local-model"
$env:LOCAL_LLM_FAST_MODEL="my-local-model"
$env:LOCAL_LLM_URL="http://127.0.0.1:8080"
$env:LOCAL_LLM_FAST_URL="http://127.0.0.1:8080"

.venv\Scripts\streamlit.exe run app.py
```

Если модель уже запущена в Ollama, LM Studio или другом совместимом сервере, укажите его OpenAI-совместимый адрес через `LOCAL_LLM_URL`. Например:

```powershell
$env:LOCAL_LLM_URL="http://127.0.0.1:1234"
$env:LOCAL_LLM_FAST_URL="http://127.0.0.1:1234"
$env:LOCAL_LLM_MODEL="имя-модели-в-сервере"
$env:LOCAL_LLM_FAST_MODEL="имя-модели-в-сервере"
.venv\Scripts\streamlit.exe run app.py
```

Обе переменные адреса указывают на один сервер, если используется одна модель. Сервер должен поддерживать OpenAI-совместимый путь `/v1/chat/completions` и структурированный JSON-ответ.

## Голосовой режим

Локальные компоненты устанавливаются отдельной командой:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup_local_voice.ps1
```

Она устанавливает Whisper для распознавания речи, Silero для определения границ реплики, LanguageTool для независимой проверки и локальную Java. Облачный голосовой режим необязателен и не нужен для основного учебного цикла.

## Запуск через Docker

Сначала один раз скачайте локальную модель:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup_local_llm.ps1
docker compose up --build
```

После запуска интерфейс доступен по адресу <http://localhost:8501>. Веса моделей не хранятся в Git из-за ограничения размера; установочный скрипт скачивает закреплённые версии и проверяет SHA-256.

## Как устроена система

```text
Ответ студента
    ↓
Предметный модуль: точные компоненты и частичный балл
    ↓
Локальная модель: два вопроса проверки понимания
    ↓
Независимая проверка правил и смысловая калибровка
    ↓
Объединение истории, задания и ответов студента
    ↓
Персональная практика и сводка пробелов группы
    ↓
SQLite: результаты, журнал вызовов и версия модели
```

## Ограничения текущего MVP

- на центральном процессоре смысловая проверка обычно занимает 10–25 секунд;
- качество проверено на небольшом экспертном наборе, а образовательный эффект требует пилота;
- произношение и акцент пока не входят в итоговый балл;
- авторизация и интеграция с университетскими учебными системами не реализованы;
- голосовой режим зависит от качества микрофона и локального распознавания.
