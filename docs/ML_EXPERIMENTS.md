# ML-эксперименты VivaTrace

## 1. Семантическая оценка Viva

### Проблема

Оценка по близости к одной эталонной фразе даёт две системные ошибки: отклоняет правильные
перефразирования и принимает ответы, содержащие одно знакомое слово без требуемого смысла.

### Решение

Каждый диагностический вопрос получает атомарную рубрику `expected_concepts`. Локальная LLM
оценивает смысл, а код независимо проверяет наличие опорных понятий и калибрует границы балла:

- полное покрытие не может получить меньше 85% только из-за перефразирования;
- частичное покрытие получает частичный, а не нулевой балл;
- неполный ответ не может получить status correct только за валидный пример без требуемого объяснения;
- нулевое покрытие не может получить больше 30%, даже если LLM ошибочно увидела знакомое слово;
- итоговые статусы вычисляются из откалиброванного балла: correct ≥ 75%, partial ≥ 30%.

Размеченный набор находится в `data/viva_semantic_benchmark.json`. Он включает правильные,
частичные, нерелевантные и adversarial-ответы по reported speech, gerund/infinitive, modal
deduction, articles и relative clauses.

```powershell
$env:PYTHONPATH='src;.'
.venv\Scripts\python.exe scripts\evaluate_viva_semantics.py `
  --output artifacts\viva_semantic_metrics.json
```

Метрики: доля попаданий в экспертный интервал, средняя дистанция до интервала, false accept,
false reject и средняя задержка. Финальный живой прогон локальной Qwen по 14 сценариям:

- попадание в экспертный интервал: **100%**;
- mean distance to band: **0.0**;
- false accepts: **0**;
- false rejects: **0**;
- средняя задержка: **16,8 секунды**.

Итерации benchmark действительно нашли дефекты: сначала false accept на `must be` вместо
`must have + V3`, затем слишком узкие синонимы `stop to do`, русских падежных форм и
варианта «that здесь нельзя». Эти случаи
сохранены в наборе как постоянные регрессии, а не удалены ради улучшения метрики.

## 2. Knowledge tracing / BKT

### Постановка

Целевая переменная — успешность следующего взаимодействия студента с тем же навыком. Сравниваются:

1. средняя успешность обучающей выборки;
2. последний наблюдаемый балл;
3. экспоненциальное среднее (EMA);
4. Bayesian Knowledge Tracing;
5. BKT с isotonic calibration.

Параметры BKT и alpha EMA выбираются только на fit-части. Isotonic regression обучается на
отдельной calibration-части. Финальные метрики считаются на студентах из test, которых не было
при настройке.

```powershell
$env:PYTHONPATH='src;.'
.venv\Scripts\python.exe scripts\evaluate_knowledge_tracing.py `
  --output artifacts\knowledge_tracing_metrics.json
```

### Метрики воспроизводимого pilot-эксперимента

Seed 42, 180 синтетических студентов, 660 test-событий:

| Модель | Brier ↓ | Log loss ↓ | ECE-10 ↓ | Accuracy |
|---|---:|---:|---:|---:|
| Global mean | 0.2402 | 0.6734 | 0.0142 | 0.6000 |
| Последний ответ | 0.2354 | 0.7358 | 0.0944 | 0.6227 |
| EMA | **0.2208** | **0.6323** | 0.0598 | 0.6470 |
| BKT | 0.2298 | 0.7030 | 0.1251 | 0.6515 |
| BKT + isotonic | 0.2278 | 0.7332 | 0.0872 | **0.6530** |

Вывод не скрывается: на синтетическом pilot EMA лучше по Brier и log loss, а сырой BKT
переуверен. Isotonic calibration улучшает Brier и ECE BKT, но не делает его лучшим по всем
метрикам. Поэтому production-интерфейс пока использует прозрачное evidence fusion, а BKT остаётся
экспериментальным кандидатом до сбора последовательностей реальных студентов.

Синтетические данные нужны только для проверки разделения выборок, подбора параметров и расчёта
метрик. Они не являются доказательством образовательного эффекта. Следующий этап — зафиксировать
протокол пилота и повторить сравнение на реальных анонимизированных последовательностях.

## 3. Grounding практических миссий

`scripts/evaluate_missions.py` проверяет положительный и отрицательный пример по
каждой из десяти тем, а с флагом `--live` дополнительно выполняет реальные ходы
локальной Qwen.

```powershell
.venv\Scripts\python.exe scripts\evaluate_missions.py --live-all `
  --output artifacts\mission_benchmark.json
```

Детерминированный контур: **20/20** — все целевые конструкции найдены в валидных
примерах и ни одна не найдена в нерелевантном ответе. Финальный живой прогон:
**11/11**, средняя задержка одного хода **11,7 секунды**. В набор входят десять
валидных тематических реплик и один нерелевантный ответ с итогом 0%.

Предыдущие итерации действительно обнаружили ложные замены `a blue tag → the blue
tag`, `plan to study → plan to decide to study` и стилистическое переписывание
валидного `a place where…`. После этого добавлены общие фильтры буквальной цитаты,
локальности/no-op исправления и предметные guardrail. Все регрессии сохранены в
тестах, включая обратные случаи с настоящими ошибками.
