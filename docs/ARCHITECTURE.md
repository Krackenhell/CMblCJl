# Архитектура VivaTrace

## Основной принцип

LLM не хранит «истину о студенте» и не выставляет непрозрачный итоговый балл. Она выполняет ограниченные задачи генерации вопроса и извлечения evidence. Состояние знаний, правила маршрутизации и групповые агрегаты представлены явными структурами данных.

## Модули

### Curriculum Graph

`data/curriculum.json` хранит навыки, prerequisite-связи и целевой mastery. В production-версии LLM может предложить карту из программы курса, но преподаватель обязан её подтвердить.

### Artifact Inspector

`src/vivatrace/artifact.py` извлекает проверяемые сигналы из работы. Он не объявляет ошибку доказанной: каждый сигнал превращается в гипотезу для viva.

### Viva Planner

`src/vivatrace/viva.py` ранжирует вопросы по двум сигналам:

1. проблемные места конкретного артефакта;
2. навыки с низким текущим mastery.

Так вопросы персонализированы по работе и модели знаний, а не генерируются как общий тест по теме.

### Evidence Evaluator

`src/vivatrace/evaluator.py` имеет два адаптера:

- `RubricEvaluator` — прозрачный детерминированный baseline;
- `OpenAICompatibleEvaluator` — опциональная LLM-оценка со structured JSON.

Результат всегда содержит `score`, `confidence`, цитату, обоснование и необязательный misconception label.

### BKT Learner Model

`src/vivatrace/bkt.py` реализует стандартную четырёхпараметрическую структуру Bayesian Knowledge Tracing:

- `prior` — вероятность исходного знания;
- `learn` — вероятность освоения после взаимодействия;
- `slip` — вероятность ошибки при знании;
- `guess` — вероятность правильного ответа без знания.

В отличие от бинарного ответа OATutor, VivaTrace интерполирует likelihood по непрерывному rubric score. Это позволяет учитывать частичное понимание в открытом ответе.

### Routing Engine

`src/vivatrace/routing.py` использует прозрачные правила:

- низкая уверенность → `human_review`;
- mastery ниже 0.45 → `repair`;
- mastery ниже цели навыка → `practice`;
- mastery выше цели → `transfer`.

Пороговые значения являются продуктовой гипотезой и должны калиброваться на пилоте.

### Cohort Intelligence

`src/vivatrace/cohort.py` агрегирует mastery и misconceptions. Политика интервенции MVP:

- пробел у 35% и более → повтор для всей группы;
- 10–35% → малая группа;
- менее 10% → индивидуальный repair.

## Production evolution

1. Подключить LMS через LTI/xAPI.
2. Хранить event log и версионировать curriculum graph.
3. Обучить misconception classifier на экспертной разметке.
4. Калибровать confidence и пороги selective prediction.
5. Добавить асинхронную голосовую viva и письменную accessibility-альтернативу.
6. Провести A/B-тест: обычный feedback против VivaTrace loop.

## Риски и меры

| Риск | Мера |
|---|---|
| LLM ошибочно оценивает ответ | Цитата-основание, confidence, human review, апелляция |
| Оценивается речь вместо знания | Предметная рубрика не содержит fluency; письменная альтернатива |
| Remediation демотивирует | Короткий repair вместо дополнительного объёма однотипных задач |
| Система стигматизирует студентов | Динамические состояния навыков вместо ярлыков «сильный/слабый» |
| Утечка персональных данных | Псевдонимизация, локальное развёртывание, агрегированный dashboard |
